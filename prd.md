一、项目定位

本前端系统围绕"关键词"维度的新闻打分与总结数据，提供三个功能模块：
	•	模块一：关键词总结 + 当日新闻打分列表
	•	模块二：新闻抓取来源可视化分析
	•	模块三：修改评分管理（需要管理员密码保护）

页面可使用 Tab 页或侧边导航栏实现模块切换。大部分功能无需身份认证，所有用户均可直接浏览，但修改评分功能需要管理员密码验证。

⸻

📘 模块一：关键词总结 + 当日新闻列表

目标：

集中展示每个关键词在某一天的总结内容，并列出当天抓取的新闻及其AI打分明细，支持关键词、日期等多维度筛选与信息预览。

数据来源：
	•	summary_news：关键词每日总结
	•	scored_news：关键词每日抓取的新闻条目及评分结果

展示内容：
	1.	关键词摘要
	•	日期
	•	关键词
	•	总结内容（支持折叠/展开）
	2.	当天新闻列表
	•	标题（点击可展开查看全文）
	•	正文预览（默认前100字）
	•	抓取来源（引用链接/站点名）
	•	AI评分（支持排序）

筛选条件：
	•	日期（起止范围）
	•	关键词（支持多选/模糊匹配）
	•	来源网站（选填）
	•	AI评分区间（如 5-0分）

接口参数设计：
	•	分页：page（页码，从1开始），pageSize（每页条数，默认20，可选10/20/50/100）
	•	排序：sortBy（排序字段，如score、date等），order（asc/desc）
	•	筛选：filters（对象结构，如{keyword:[], dateRange:[], source:[], scoreRange:[] }）
	•	所有接口均返回JSON格式数据
	•	无需鉴权，但建议后端实现基础限流防护

交互功能：
	•	总结部分与新闻列表按关键词组合展示
	•	每页分页控制
	•	页面加载快速、响应顺畅
	•	去除导出功能

⸻

📊 模块二：新闻抓取来源可视化分析

目标：

从"关键词 + 日期"两个维度，分析各网站的新闻抓取量分布情况，识别活跃来源、热点关键词、采集偏好。

数据来源：
	•	news_website：网站基本信息（如名称、链接）
	•	news_source_stats：每天每个关键词从各网站抓取的数量统计

展示内容：
	•	各关键词在选定日期范围内的抓取总量按网站分布（柱状图、表格）
	•	单个关键词在多个网站上的对比趋势（时间轴折线图）
	•	网站总抓取量排行榜（TOP10）

筛选条件：
	•	日期范围
	•	关键词（单选或多选）

交互功能：
	•	可视化图表（柱状图、折线图）
	•	图表下方附带原始数据表格，表格支持排序、筛选、分页
	•	图表和表格联动（如点击图表某项，表格自动筛选对应数据）
	•	图表可切换：按关键词聚合 or 按网站聚合
	•	去除导出功能

⸻

📝 模块三：修改评分管理

目标：

为系统管理员提供修改新闻AI评分的功能，用于质量控制和评分校正，确保评分系统的准确性和可靠性。

数据来源：
	•	scored_news：新闻评分数据表，包含所有新闻的AI评分信息

访问控制：
	•	管理员密码保护：需要输入正确的管理员密码才能访问
	•	会话管理：验证成功后30分钟内无需重复输入密码
	•	密码配置：密码存储在环境变量中，便于管理和修改

展示内容：
	1.	筛选控件
	•	关键词选择器（下拉列表）
	•	日期选择器
	•	支持实时数据获取

	2.	新闻列表展示
	•	新闻标题（可点击跳转到原始页面）
	•	搜索关键词信息
	•	当前AI评分状态
	•	完整新闻正文内容

	3.	评分修改功能
	•	评分下拉选择器（0-5分，带描述）
	•	修改按钮
	•	修改状态反馈
	•	成功提示信息

交互功能：
	•	密码验证界面（现代化设计）
	•	实时评分修改（即时生效）
	•	加载状态显示
	•	错误处理和用户提示
	•	退出登录功能

技术特性：
	•	前端密码验证（适合内部使用）
	•	本地会话状态管理
	•	响应式界面设计
	•	数据库实时更新
	•	API参数验证和错误处理

📰 模块四：AI新闻周报生成（/report 页面）

目标：

为指定关键词（如“江苏省国资委”）在选定日期范围内，自动生成结构化的AI新闻周报，支持导出图片和PDF，适合对外展示和归档。

主要功能：
1. 日期选择器默认行为：
   - 默认开始日期为“本周一”，结束日期为“今天”。
   - 每周第一天为周一（即周日时回溯到上周一）。
2. 新闻筛选：
   - 默认只显示AI评分3分及以上的高质量新闻。
   - 支持切换显示全部新闻（包括0-2分低分新闻）。
   - 支持全选/取消全选新闻。
3. 周报生成：
   - 用户选择新闻后，点击“生成周报”按钮，自动调用大模型生成结构化Markdown格式的周报。
   - 支持用户自定义补充提示（Prompt）。
   - 支持二轮修改：用户可输入修改意见，AI流式输出优化后的新版本。
4. 大模型自动切换与容错：
   - 默认优先调用DeepSeek R1模型。
   - 若新闻数量过多导致DeepSeek返回token超限报错，系统自动切换到Google Gemini 2.5 Pro大模型生成总结，无需用户干预。
   - 前端有友好提示，明确告知“已自动切换到Gemini 2.5 Pro”。
5. 导出与分享：
   - 支持一键导出周报为高质量图片（1080px宽，科技感模板）。
   - 支持导出为PDF，带有AI科技感样式。
6. 体验细节：
   - AI思考过程默认展开，内容左对齐，便于理解生成逻辑。
   - 支持查看本次生成的Prompt和模型信息。
   - 生成过程支持流式输出，失败时可一键重试。

数据来源：
   - scored_news：新闻原始内容及AI评分
   - 关键词固定为“江苏省国资委”（后续可扩展）

接口参数设计：
   - 详见后端 /api/generate-report，支持流式和非流式两种模式，自动fallback到Gemini。

交互功能：
   - 现代化、科技感UI，所有内容左对齐，支持大屏展示
   - 主要操作按钮有加载状态和禁用逻辑
   - 错误和切换提示清晰友好

三、技术和设计建议

1. 架构建议
	•	后端接口基于 RESTful API，返回 JSON 数据；
	•	前端建议使用：
	•	Vue3 + Element Plus 或 React + shadcn/ui（性能轻、开发快）
	•	Axios 请求接口
	•	若数据量较大，需支持后端分页
	•	数据库连接信息存放于 .env 文件中

2. 性能优化
	•	新闻正文字段预览默认显示前 100 字，点展开再加载全文；
	•	总结内容支持按需加载，默认折叠状态；
	•	数据按需请求（关键词和日期选择后再拉取结果）
	•	表格和图表数据量大时，前端需支持虚拟滚动或懒加载

3. 错误处理与提示
	•	接口异常：页面顶部弹出统一错误提示（如"数据加载失败，请稍后重试"）
	•	数据为空：在表格/图表区域显示"暂无数据"
	•	加载中：显示 loading 动画或骨架屏
	•	所有提示文案风格简洁明了

4. UI与交互细节
	•	整体风格现代、科技感，配色建议以蓝色、灰色为主
	•	主要按钮、Tab、图表等有适度动画和过渡效果，提升体验
	•	不需要中英文切换
	•	不需要移动端适配

5. 统计口径说明
	•	所有统计均以"fetchdate"字段为准，仅区分日期（不区分具体时间）
	•	如有统计口径疑问，可在页面适当位置（如图表下方）增加说明

6. 其他
	•	关键词和网站信息由后端维护，无需前端管理界面
	•	暂不考虑未来扩展和自定义报表
