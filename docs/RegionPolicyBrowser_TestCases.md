# 地域政策浏览功能测试案例

## 测试概述
本文档包含"地域政策浏览"功能的10个测试案例，覆盖地域准确性、评分准确性、新闻数量准确性等核心功能。

---

## 测试案例 1：全国级地域查询准确性

**测试目标**：验证点击"全国政策"显示的新闻是否准确

**前置条件**：数据库中存在 `region='全国'` 的新闻数据

**测试步骤**：
1. 访问 `/policy/regions` 页面
2. 在左侧地域树中点击"全国政策"
3. 查看右侧新闻列表

**预期结果**：
- 列表中只显示 `region='全国'` 的新闻
- 新闻标题、来源、日期、评分、地域字段完整显示
- 数量统计与实际返回记录数一致

**验证SQL**：
```sql
SELECT COUNT(*) FROM scored_news WHERE keyword='公积金' AND region='全国';
```

---

## 测试案例 2：直辖市查询准确性

**测试目标**：验证直辖市（北京、上海等）新闻查询是否准确

**前置条件**：数据库中存在北京市、上海市相关新闻

**测试步骤**：
1. 点击左侧"上海市"
2. 观察新闻列表
3. 点击"北京市"观察切换

**预期结果**：
- 上海市显示 `region` 包含"上海市"或"上海"的新闻
- 切换到北京后，列表正确刷新为北京相关新闻
- 不会显示其他直辖市的新闻

**边界情况**：
- 测试 `region='上海'`（不带"市"）的情况
- 测试 `region='上海市'`（带"市"）的情况
- 测试多地域情况如 `region='上海|北京'`

---

## 测试案例 3：多地域字段解析准确性

**测试目标**：验证 `region` 字段中包含多个地域（|分隔）时的处理

**前置条件**：数据库中存在 `region='湖南|娄底'`、`region='湖南|长沙'` 等数据

**测试步骤**：
1. 展开湖南省
2. 点击"娄底"
3. 查看返回的新闻列表

**预期结果**：
- 娄底的新闻列表包含 `region='娄底'` 和 `region='湖南|娄底'` 的新闻
- 长沙的新闻列表不包含娄底的新闻
- 省级视图显示该省所有城市的新闻总和

**验证SQL**：
```sql
-- 娄底应显示的新闻
SELECT COUNT(*) FROM scored_news
WHERE keyword='公积金' AND (
  region = '娄底' OR
  region LIKE '%|娄底%' OR
  region LIKE '%娄底|%'
);
```

## 测试案例 4：评分准确性验证

**测试目标**：验证评分显示是否准确，4分和5分颜色是否区分

**前置条件**：数据库中存在 score=4 和 score=5 的新闻

**测试步骤**：
1. 点击任意地域，如"全国"
2. 观察列表中评分列的显示
3. 验证不同评分的颜色

**预期结果**：
- 4分显示为浅绿色背景（#dcfce7）+ 深绿文字
- 5分显示为琥珀金背景（#d97706）+ 白色文字
- 评分数值与数据库中的 `score` 字段一致

**颜色验证**：
| 评分 | 背景色 | 文字色 |
|-----|--------|--------|
| 0-1 | #fee2e2 | #991b1b |
| 2-3 | #fef3c7 | #92400e |
| 4 | #dcfce7 | #166534 |
| 5 | #d97706 | #ffffff |

## 测试案例 5：新闻数量统计准确性

**测试目标**：验证地域树中的数量统计与实际查询结果一致

**前置条件**：页面已加载地域树

**测试步骤**：
1. 记录"全国政策"旁显示的数量（如53条）
2. 点击"全国政策"
3. 观察新闻列表底部的分页信息

**预期结果**：
- 地域树显示的数量 = 实际新闻总数
- 分页显示"共 X 页"与总数计算一致（每页20条）

**验证方法**：
```javascript
// 检查总数是否匹配
const treeCount = document.querySelector('.national .region-count').textContent;
const tableTotal = document.querySelector('.header-badge').textContent;
expect(treeCount).toBe(tableTotal);
```

## 测试案例 6：时间范围筛选准确性

**测试目标**：验证日期选择器筛选功能

**前置条件**：数据库中存在不同日期的新闻

**测试步骤**：
1. 设置起始日期为昨天，结束日期为今天
2. 观察地域树数量变化
3. 点击某个地域查看新闻列表
4. 清空日期重新筛选

**预期结果**：
- 设置日期后，地域树重新加载，数量更新
- 新闻列表只显示日期范围内的记录
- 分页重置为第一页

**边界情况**：
- 选择无数据的日期范围，显示空状态
- 起始日期 > 结束日期的处理

## 测试案例 7：省级新闻查询（含下属城市）

**测试目标**：验证点击省份显示该省所有新闻（含下属城市）

**前置条件**：江苏省下有南京、苏州、扬州等城市新闻

**测试步骤**：
1. 展开"江苏省"
2. 点击"江苏省"（不是下属城市）
3. 查看新闻列表

**预期结果**：
- 显示所有江苏省相关新闻，包括：
  - `region='江苏省'` 的省级新闻
  - `region='江苏'` 的简写形式
  - 下属城市如南京、苏州、扬州的新闻
- 不包含其他省份的新闻

**验证SQL**：
```sql
-- 江苏省应包含
SELECT * FROM scored_news WHERE keyword='公积金' AND (
  region = '江苏省' OR
  region = '江苏' OR
  region LIKE '%南京%' OR
  region LIKE '%苏州%' OR
  -- ... 其他江苏城市
);
```

## 测试案例 9：国外地域过滤

**测试目标**：验证国外地域新闻被正确过滤

**前置条件**：数据库中存在 `region='美国'`、`region='日本'` 等国外数据

**测试步骤**：
1. 查看地域树列表
2. 检查是否包含国外地域

**预期结果**：
- 地域树中不显示任何国外地域
- 查询结果中不包含国外新闻

**代码验证**：
```javascript
// foreignRegions 列表包含：美国、日本、韩国、英国、德国、法国、意大利、加拿大、澳大利亚
const foreignRegions = ['美国', '日本', '韩国', '英国', '德国', '法国', '意大利', '加拿大', '澳大利亚'];
// 这些地域不应出现在 regionTree 中
```

## 测试案例 10：平均评分计算准确性

**测试目标**：验证地域树中显示的"均分"计算正确

**前置条件**：某地域有多条新闻，评分各不相同

**测试步骤**：
1. 查看"北京市"显示的均分
2. 点击北京市查看新闻列表
3. 手动计算列表中所有新闻的平均分

**预期结果**：
- 均分 = 所有新闻评分之和 / 新闻数量
- 保留两位小数
- 考虑多地域新闻的重复计算问题（同一条新闻在多个地域的统计）

**计算公式**：
```javascript
avgScore = (score1 + score2 + ... + scoreN) / N
// 保留两位小数
avgScore = avgScore.toFixed(2)
```
