# 实施计划：周报政策对比模块

## 1. 后端 API 开发 (`server.cjs`)
### 新增提示词配置支持
*   更新提示词读取/保存逻辑，新增支持 `Policy Extraction Prompt`（周报抽取提示词）。
*   此提示词将用于从周报文本中提取结构化的 JSON 政策数据。

### 新增核心功能接口
1.  **`POST /api/policy/extract`**
    *   **功能**：接收周报内容，调用 LLM（使用“抽取提示词”）将其转换为标准 JSON 格式。
    *   **输入**：周报文本内容。
    *   **输出**：结构化 JSON 数据。
2.  **`POST /api/policy/compare`**
    *   **功能**：接收“抽取出的周报政策 JSON”和“现行政策 JSON”，调用 LLM（使用“对比提示词”）生成对比分析。
    *   **输入**：`extractedJson`, `currentPolicyJson`。
    *   **输出**：Markdown 格式的对比报告。

## 2. 前端配置页面更新 (`ReportConfig.jsx`)
*   在“周报参数”页面新增 **"周报抽取 Prompt"** 标签页。
*   允许用户编辑和保存用于从周报中提取信息的提示词模板。

## 3. 前端对比模块开发 (`WeeklyComparison.jsx`)
采用**分步向导**模式设计界面：

### 第一步：选择周报
*   自动加载“历史周报”列表。
*   **默认筛选**：自动应用 `keyword=公积金` 过滤条件。
*   **交互**：用户点击卡片选择一份目标周报。

### 第二步：智能处理（自动化）
*   **抽取**：调用 `/api/policy/extract` 将选中周报转化为 JSON。
*   **对比**：自动获取最新版“现行政策”，调用 `/api/policy/compare` 生成对比结果。
*   **状态展示**：显示清晰的进度条或加载动画（"正在抽取关键信息..." -> "正在与现行政策比对..."）。

### 第三步：结果展示与导出
*   **现代简约模板**：
    *   设计一个专门的 HTML 容器，样式参考“卡片式”布局。
    *   使用 `react-markdown` 渲染对比结果，通过 CSS 强化“新旧政策”的视觉差异（如：左侧旧政策灰底，右侧新政策高亮）。
*   **图片导出**：
    *   使用 `html2canvas` 将对比结果区域生成为高清图片。
    *   **手机适配**：确保生成的图片宽度适中，字体清晰，适合在手机屏幕上阅读。

## 4. 样式优化 (`WeeklyComparison.css`)
*   设计响应式布局，确保在生成图片时排版不乱。
*   定义清晰的排版层级：标题、对比项、总结分析。
