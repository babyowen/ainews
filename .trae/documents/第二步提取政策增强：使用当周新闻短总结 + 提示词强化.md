## 目标
- 第二步“提取政策”产出的政策内容更具体（含额度/条件/期限/对象/材料/流程等细节）。
- “选择周报”阶段新增一种数据来源：不选已生成周报文本，而是**直接读取数据库当周 keyword=公积金 且 score>=4 的全部新闻**，并使用每条新闻的 **short_summary（约500字）** 来拼接“当周全部新闻”，再用于政策提取。

## 现状定位
- 第二步提取接口支持两种输入：`reportId` 或 `reportContent`（[server.cjs](file:///Users/babyowen/Documents/GitHub/ainews/server.cjs#L2116-L2218)）。
- 数据库新闻筛选接口已存在：`GET /api/weekly-news?keyword&startDate&endDate&minScore=4`，返回 `short_summary`（[server.cjs](file:///Users/babyowen/Documents/GitHub/ainews/server.cjs#L457-L495)）。
- 当前“选择周报”仅能选 `weekly_reports`（[WeeklyComparison.jsx](file:///Users/babyowen/Documents/GitHub/ainews/src/pages/PolicyComparison/WeeklyComparison.jsx#L509-L549)）。
- 当前“提取政策”提示词与最新政策模板字段存在不一致（提示词提到“核心内容/依据文档/执行时间”，而模板实际是“明细项/内容/依据文件/子项”等；见最新政策JSON [policy_20260125035220.json](file:///Users/babyowen/Documents/GitHub/ainews/config/policies/policy_20260125035220.json)）。这会降低提取质量。

## 改动方案
### 1) 强化第二步“提取政策”提示词（提高细节密度）
- 更新 [policy_prompts.md](file:///Users/babyowen/Documents/GitHub/ainews/config/policy_prompts.md) 的 **Policy Extraction Prompt**：
  - 将字段约束改为与模板一致（政策领域/领域名称/政策类别/类别名称/政策明细；明细项/内容/依据文件/子项）。
  - 明确要求在“内容”里尽量结构化写全：
    - 适用对象、门槛条件、额度/比例、期限/时间点、次数限制、材料/流程、限制/例外。
  - 要求“不要过度合并”：同一类别下不同城市/不同条款拆分为不同“政策明细”。
  - 对来源要求更强：`依据文件` 用 “新闻周报-{城市}-{新闻标题或来源}” 形式；如无法确定城市也要标注来源标题。
  - 在输入部分把 `{report}` 的语义从“周报正文”扩展为“周报正文或当周全部新闻摘要集合”，让模型能理解两种输入。

### 2) 第一步新增数据源：周报 vs 当周全部新闻（公积金>=4）
- 在 [WeeklyComparison.jsx](file:///Users/babyowen/Documents/GitHub/ainews/src/pages/PolicyComparison/WeeklyComparison.jsx) 的 Step 1：
  - 增加一个选择器（Tab/Segment）：
    - 选项A：使用“已生成周报”（现状保持不变）。
    - 选项B：使用“当周全部新闻（公积金、评分>=4）”。
  - 选项B的交互：
    - 仍通过“周报卡片”选择时间范围（start_date/end_date），用于查询当周新闻。
    - 请求 `GET /api/weekly-news?keyword=公积金&startDate&endDate&minScore=4`。
    - 以 `/report` 页同款风格展示新闻列表（标题 + short_summary），并默认全选；允许用户取消勾选少量无关新闻。
    - 拼接“当周全部新闻”文本：每条包含【标题】【来源/日期/评分】【short_summary】（并对缺少 short_summary 的新闻回退 `content` 并截断到约500字）。

### 3) 第二步提取调用改为按数据源传参
- 若选择“已生成周报”：维持 `POST /api/policy/extract { reportId }`。
- 若选择“当周全部新闻”：改为 `POST /api/policy/extract { reportContent }`，其中 reportContent 为上一步拼接的“当周全部新闻”文本。
- 同步更新“提取阶段 - 实时预览”的 preview prompt 请求：
  - 周报模式传 `reportId`
  - 新闻模式传 `reportContent`

### 4) 开发期调试增强
- 在第二步页面增加一个可折叠区域，展示“当周全部新闻（拼接后）”的原始文本，方便你判断提取输入是否足够具体。
