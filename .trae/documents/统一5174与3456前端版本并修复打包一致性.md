## 结论与原因
- 存在两个服务同时运行：
  - 前端开发服务 `http://localhost:5174`（由 `vite` 启动，端口配置在 `vite.config.js:8-15`）。
  - 后端/API 服务 `http://localhost:3456`（由 `server.cjs` 启动，并同时静态托管 `dist` 构建产物，参见 `server.cjs:2296-2304`）。
- `http://localhost:3456` 显示的是此前打包后的 `dist` 版本；`http://localhost:5174` 显示的是当前 `src` 源码版本。
- 我已检索到当前 `src` 中未包含您提到的“多 prompt 选择”“短总结选择”等 UI 文案或控件，因此 5174 的页面缺少这些新功能；而 3456 的页面是旧构建中包含这些功能的打包产物。
- 风险：如果现在直接执行 `npm run build`，将会按照当前 `src` 源码重新生成 `dist`，从而覆盖掉 3456 正在展示的那份“正确版本”。

- `package.json:7-12` 指定 `npm run dev` 并发启动 API(默认为 `3456`) 与 Vite(默认为 `5174`)。
- `vite.config.js:8-15` 将前端端口固定为 `5174`，并将 `/api` 代理到 `http://localhost:${API_PORT||3456}`。
- `server.cjs:2296-2304` 在后端服务上托管 `dist`，因此 3456 会呈现打包后的前端。
- `dist/index.html:14-16` 是当前线上预览的入口文件（包含字体等），`index.html:7-13,16-18` 是开发态入口（加载 `src/main.jsx`）。
