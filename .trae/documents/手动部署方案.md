# 手动部署方案 (Copy & Paste)

这是一个最精简的手动更新流程。你只需要复制 **4 个文件** 的内容到服务器对应的文件中，然后执行 **2 个命令** 即可。

## 1. 替换文件 (共 4 个)
请在服务器上找到对应的文件，用文本编辑器（如 `vim` 或 `nano`）打开，清空原内容，粘贴新内容。

### (1) 核心配置：`src/config/keywords.js`
> **作用**：让前端所有页面的下拉菜单出现“潜在招标客户”。

### (2) 后端逻辑：`server.cjs`
> **作用**：让后端统计接口支持新关键词的聚合。

### (3) Prompt 配置：`config/keyword-prompts.json`
> **作用**：为新关键词提供默认的周报生成模板。

### (4) 样式修正：`src/index.css`
> **作用**：修复页面背景黑屏问题，确保显示正常。

---

## 2. 执行命令 (必须执行)
文件替换完成后，必须在服务器的项目根目录下运行以下命令，否则修改**不会生效**。

### (1) 重新构建前端
```bash
npm run build
```
*解释：这一步会将 `src/` 下的新代码编译成浏览器可运行的 `dist/` 文件。如果不跑这个，前端页面依然是旧的。*

### (2) 重启后端服务
根据你的服务器运行方式选择一种：

**方式 A：如果你使用 PM2 (推荐)**
```bash
pm2 reload all
```

**方式 B：如果你是直接用 node 运行**
先按 `Ctrl+C` 停止当前进程，然后重新运行：
```bash
API_PORT=3456 node server.cjs
```
*(注意端口号按你实际情况修改，如果服务器用的是 80 或其他端口)*

---

## 3. 验证
访问你的服务器网站，刷新页面：
1. 关键词下拉列表是否有“潜在招标客户”？
2. 页面背景是否正常（白色/浅色）？
